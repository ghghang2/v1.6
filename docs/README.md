## How to Use This Documentation

### For New Contributors
1. Start with [ARCHITECTURE.md](./ARCHITECTURE.md) to understand the system
2. Check [IMPLEMENTATION_PLAN.md](./IMPLEMENTATION_PLAN.md) for current priorities
3. Review [TESTING_GUIDE.md](./TESTING_GUIDE.md) before running tests

### For Active Development
1. Update [IMPLEMENTATION_PLAN.md](./IMPLEMENTATION_PLAN.md) with progress
2. Keep test coverage documented in [TESTING_GUIDE.md](./TESTING_GUIDE.md)
3. Add architectural decisions to [ARCHITECTURE.md](./ARCHITECTURE.md)

### For Session Handoffs
1. Review [IMPLEMENTATION_PLAN.md](./IMPLEMENTATION_PLAN.md) for current status
2. Check each component's specific documentation for details
3. Update progress before ending session

## Document Format

Each document follows this structure:
- **Overview**: What is this about?
- **Current Status**: What's done, what's in progress, what's pending
- **Implementation Details**: Technical specifics
- **References**: Links to external resources, papers, or code
- **Next Steps**: Immediate actions required

## Maintaining This Documentation

- Keep documents concise and scannable
- Use tables for status tracking
- Include links to relevant code files
- Update status after each session
- Archive old versions in `docs/archive/` before major reorganizations
